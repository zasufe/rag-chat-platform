# RAG 知识库 - 提示词优化方案

## 当前问题分析

### 问题
- 知识库检索强制使用，即使没有相关内容也限制模型发挥
- 提示词过于强调"优先基于知识库"，限制了模型能力
- 没有区分"知识库有相关内容"和"知识库无相关内容"的场景

### 目标
- **知识库有相关内容**：增强回答，提供准确信息
- **知识库无相关内容**：基于模型自身能力自由回答
- **重点**：给予模型自身能力进行回答

---

## 优化方案

### 方案 1：前端请求参数优化

```json
{
  "messages": [...],
  "code": "...",
  "text": "你的问题",
  "kb_id": "xxx",

  "top_k": 3,
  "kb_mode": "enhance"  // 新增：enhance(增强) | strict(严格) | free(自由)

```

### 方案 2：后端提示词优化

修改 `build_rag_prompt` 方法，使用更灵活的提示词策略：

```python
def build_rag_prompt(
    self,
    code: str,
    query: str,
    references: List[Dict[str, Any]] = None,
    history: List[Dict[str, str]] = None,
    kb_mode: str = "enhance"
) -> List[Dict[str, str]]:
    """构建 RAG 提示词 - 增强模式"""
    
    # 构建上下文
    context_text = ""
    if references:
        context_text = "\n\n".join([
            f"[参考文档 {i+1}] 来源：{ref.get('metadata', {}).get('filename', '未知')}\n{ref['content']}"
            for i, ref in enumerate(references)
        ])
    
    # 构建系统提示 - 增强模式
    if context_text:
        system_prompt = f"""你是一个专业的 AI 助手。

## 参考资料
以下资料**可能**与用户问题相关，供你参考：

{context_text}

## 回答指南
1. **如果参考资料与问题相关**：结合资料内容，提供更准确、详细的回答
2. **如果参考资料与问题无关**：基于你的专业知识和能力，自由回答用户问题
3. **优先保证回答质量**：资料只是辅助，你的专业知识同样重要
4. **不必强制引用**：只有当资料确实有用时才参考，不需要时可以忽略
"""
    else:
        system_prompt = """你是一个专业的 AI 助手，基于你的专业知识和能力回答用户问题。"""
    
    if code:
        system_prompt += f"""

## 当前页面代码上下文：
```
{code}
```
"""
    
    # 构建消息列表
    messages = [{"role": "system", "content": system_prompt}]
    
    # 添加历史消息
    if history:
        messages.extend(history[-10:])
    
    # 添加当前问题
    messages.append({"role": "user", "content": query})
    
    return messages
```

### 模板 1：增强模式（推荐）

```
你是一个专业的 AI 助手。

## 参考资料
以下资料**可能**与用户问题相关，供你参考：

[参考文档 1] 来源：xxx

## 回答指南
1. **如果参考资料与问题相关**：结合资料内容，提供更准确、详细的回答
2. **如果参考资料与问题无关**：基于你的专业知识和能力，自由回答用户问题
3. **优先保证回答质量**：资料只是辅助，你的专业知识同样重要
4. **不必强制引用**：只有当资料确实有用时才参考，不需要时可以忽略

## 当前问题
用户问题：{query}
```

### 模板 2：自由模式（完全依赖模型）

```
你是一个专业的 AI 助手，基于你的专业知识和能力回答用户问题。

{如果有代码上下文}
## 代码上下文
```
{code}
```

请回答用户问题：{query}
```

```python
"""
```

## 测试用例

### 测试 1：知识库有相关内容

**用户问题**: "系统功能需求有哪些？"  
**知识库**: 有需求文档  
**预期**: 结合知识库内容，详细列出功能需求

### 测试 2：知识库无相关内容

**用户问题**: "这段代码有什么安全漏洞？"  
**知识库**: 无相关代码分析内容  
**预期**: 基于模型自身能力，分析代码漏洞

### 测试 3：混合场景

**用户问题**: "如何进行固件安全分析？"  
**知识库**: 有部分相关文档  
**预期**: 结合知识库 + 模型专业知识，全面回答

## 关键改进点

| 改进项 | 之前 | 优化后 |
|--------|------|--------|
| **提示词语气** | "必须基于知识库" | "可以参考知识库" |
| **无相关内容时** | "明确告知用户" | "自由发挥专业能力" |
| **引用要求** | "必须引用来源" | "灵活引用" |
| **模型定位** | "知识库复读机" | "专业助手 + 知识增强" |

---

**核心思想**: 知识库是**增强**模型能力的工具，不是**限制**模型发挥的枷锁。
